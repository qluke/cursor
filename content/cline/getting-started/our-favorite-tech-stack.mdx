### 快速上手

1. 安装开发必备工具：
    - 参考我们的[开发必备工具安装指南](https://docs.cline.bot/getting-started/getting-started-new-coders/installing-dev-essentials)
2. 配置 Cline 记忆库：
    - 按照[记忆库设置说明](https://docs.cline.bot/improving-your-prompting-skills/custom-instructions-library/cline-memory-bank)操作
    - 在项目根目录创建空的 `cline_docs` 文件夹
    - 在 `cline_docs` 文件夹下新建 `projectBrief.md`（见下方示例）
    - 告诉 Cline “初始化记忆库”
3. 添加推荐技术栈配置：
    - 创建 `.clinerules` 文件（见下方模板）
    - 其余交给 Cline！



项目简介示例

```
# Project Brief

## Overview
Building a [type of application] that will [main purpose].

## Core Features
- Feature 1
- Feature 2
- Feature 3

## Target Users
[Describe who will use your application]

## Technical Preferences (optional)
- Any specific technologies you want to use
- Any specific requirements or constraints
```

### .clinerules 模板

```
# Project Configuration

## Tech Stack
- Next.js 14+ with App Router
- Tailwind CSS for styling
- Supabase for backend
- Vercel for deployment
- GitHub for version control

## Project Structure
/src
  /app         # Next.js App Router pages
  /components  # React components
  /lib         # Utility functions
  /types       # TypeScript types
/supabase
  /migrations  # SQL migration files
  /seed        # Seed data files
/public        # Static assets

## Database Migrations
SQL files in /supabase/migrations should:
- Use sequential numbering: 001, 002, etc.
- Include descriptive names
- Be reviewed by Cline before execution
Example: 001_create_users_table.sql

## Development Workflow
- Cline helps write and review code changes
- Vercel automatically deploys from main branch
- Database migrations reviewed by Cline before execution

## Security
DO NOT read or modify:
- .env files
- **/config/secrets.*
- Any file containing API keys or credentials
```

### 其他须知


使用 Git & GitHub

Git 帮你追踪代码变更、协作开发。常用命令如下：

**日常开发**

```
# Save your changes (do this often!)
git add .                                    # Stage all changed files
git commit -m "Add login page"              # Save changes with a clear message

# Share your changes
git push origin main                        # Upload to GitHub
```

**常见工作流**

1. **每天开始**：拉取最新代码
    
```
bashCopygit pull origin main                     # Download latest code
```

2. **开发中**：定期保存进度
    
```
bashCopygit add .
git commit -m "Clear message about changes"
```    
3. **每天结束**：推送进度
    
```
bashCopygit push origin main                     # Upload to GitHub
```    

**最佳实践**

- 经常提交，写清晰的提交信息
- 开始新工作前先拉取最新代码
- 完成后及时推送与他人共享
- 用 `.gitignore` 忽略敏感文件

<Callout title="小贴士">
Vercel 会在你推送到 main 分支时自动部署！
</Callout>


环境变量

- 开发时将密钥存储在 `.env.local`
- 生产环境在 Vercel 项目设置中添加
- 切勿将 `.env` 文件提交到 Git
