# 概述

记事本是 Cursor 中功能强大的上下文共享工具，可在作曲家和聊天互动之间架起一座桥梁。可将其视为增强型参考文档，它超越了`.cursorrules` 的功能，允许您为开发工作流程创建可重复使用的上下文。

记事本是思想、规则和文档的集合，可以

- 在开发环境的不同部分之间共享
- 使用`@`语法引用
- 使用文件附件进行增强
- 用作各种开发场景的动态模板

## 开始使用

1. 单击记事本部分中的 "+"按钮
2. 给记事本起一个有意义的名字
3. 以与在作曲家或聊天工具中相同的方式添加内容、上下文、文件和其他相关信息。
4. 使用`@`在作曲家或聊天工具中进行引用

# 主要功能

- **上下文共享**：在作曲家和聊天工具之间无缝共享上下文
- **文件附件**：附加文档和参考文件（在`.cursorrules` 中无法实现）
- **动态引用**：使用`@`提及链接到其他资源
- **灵活的内容**：以适合您需要的方式编写和组织信息

# 常见用例

1.  **动态模板生成**
    - 为常见代码模式创建模板
    - 存储特定于项目的脚手架规则
    - 在整个团队中保持一致的代码结构

2.  **架构文档**
    - 前台规范
    - 后端设计模式
    - 数据模型文档
    - 系统架构指南

3.  **开发指南**
    - 编码标准
    - 项目特定规则
    - 最佳实践
    - 团队惯例

## 常见问题

### 我应该在记事本上写些什么？

记事本适用于

- 项目架构决策
- 开发指南和标准
- 可重复使用的代码模板
- 需要经常参考的文档
- 团队特定的惯例和规则

### 哪些内容不应写入记事本？

避免将记事本用于以下用途

- 临时笔记或零散工作
- 属于版本控制（如 git）的信息
- 敏感数据或凭证
- 经常变化的高波动性信息

### 我是否应该遵循特定的格式或结构？

虽然记事本可以灵活使用，但我们建议

- 使用清晰的标题和章节
- 在相关处加入示例
- 保持内容的重点突出和条理清晰
- 使用标记符格式以提高可读性
- 必要时添加相关附件
