自主的人工智能编码代理，可独立探索、规划和执行复杂的代码库变更，并提供完整的工具

代理是 Cursor 的默认模式，也是最自主的模式，旨在以最少的指导处理复杂的编码任务。它启用了所有[工具](https://docs.cursor.com/chat/tools)，可自主探索代码库、阅读文档、浏览网页、编辑文件和运行终端命令，从而高效地完成任务。

## 代理功能

## 自主运行

独立探索代码库，识别相关文件并进行必要的修改

## 全面工具访问

使用所有可用工具搜索、编辑、创建文件和运行终端命令

## 上下文理解

全面了解项目结构和依赖关系

## 多步骤规划

将复杂任务分解为易于管理的步骤并按顺序执行

请务必阅读[聊天概述](https://docs.cursor.com/chat/overview)，进一步了解 Cursor 模式的工作原理。

## 代理的工作流程

与人类一样，Agent 模式遵循系统化的方法来完成任务：

1

🧠 理解请求

代理模式会分析您的请求和代码库的上下文，以充分理解任务要求和目标。

2

🔍 探索代码库

代理可能会搜索您的代码库、文档和网络，以确定相关文件并了解当前的实现情况。

3

计划更改

根据分析结果，代理将任务分解为更小的步骤，并计划更改，同时从可用的上下文中学习。

4

🔧 执行更改

代理会根据计划在整个代码库中对代码进行必要的修改，还可能建议使用新库、运行终端命令或在 Cursor 之外执行其他步骤。

5

🔍 验证结果

代理将通过[应用](https://docs.cursor.com/chat/apply)更改来确认更改是否正确。如果它发现任何问题或衬线错误（在语言支持的情况下），它将尝试修复它们。

6

任务完成

一旦代理感到满意，它就会返回并总结所做的更改。

代理模式会在进行更改前创建检查点，以便在需要时进行恢复。

了解有关[检查点](https://docs.cursor.com/chat/overview#checkpoints)的更多信息。

## 使用案例示例

为了达到最佳效果，请具体说明您想要完成的任务，但不必指定代理应如何处理任务。

```
"Add a dark mode toggle to my React application"

```

代理模式将

1. 确定应用程序的样式设计方法
2. 添加必要的主题状态管理
3. 创建切换组件
4. 更新相关组件的样式
5. 适当时添加持久性

```
"Optimize my API calls to implement caching and reduce redundant requests"

```

代理模式将

1. 查找所有 API 调用位置
2. 实施适当的缓存策略
3. 添加缓存失效逻辑
4. 更新组件以使用缓存数据

## 配置选项

- **选择模式**：为代理模式预选模型
- **编辑按键绑定**：设置代理模式的按键绑定
- **切换可用工具**：切换可用工具
- **启用自动运行和自动修复错误**：启用自动运行和自动修复错误（阅读有关[工具设置的](https://docs.cursor.com/chat/tools#advanced-options)更多信息）

使用启用自动运行功能的代理模式时，在确认主要修改之前，请仔细审查建议的修改。
