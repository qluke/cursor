只需点击一下鼠标，Cursor 就能帮你为更改生成有意义的提交信息。下面介绍如何使用此功能：

1. 将要提交的文件分级
2. 打开侧边栏中的 Git 标签
3. 查找提交信息输入框旁边的火花（✨）图标
4. 点击火花图标，根据暂存的更改生成提交信息

生成的提交信息将基于暂存文件中的变更和版本库的 git 历史记录。这意味着 Cursor 会分析您当前的更改和之前的提交信息，以生成与上下文相符的信息。Cursor 会从你的提交历史中学习，这意味着如果你使用类似于 "[常规提交"（Conventional Commits](https://www.conventionalcommits.org/)）这样的约定，生成的信息也会遵循相同的模式。
