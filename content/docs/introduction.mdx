Cursor 是一款功能强大的人工智能代码编辑器，可增强您的开发工作流程。[安装](https://docs.cursor.com/get-started/installation)后，您就可以使用这些核心功能，它们可以无缝协作，提高您的工作效率：

- **AI 驱动的代码自动补全**功能可理解你的代码库并提供上下文感知建议
- **对话界面**，可通过询问、编辑和代理模式探索、理解和修改代码
- 处理复杂开发任务的**智能工具**

## 开始使用

开始探索 Cursor 的人工智能功能：

- **Tab**：按`Tab 键`智能完成代码
- **CMD-K**：使用`Cmd/Ctrl + K`进行内联代码编辑
- **聊天**：使用`⌘I`打开具有询问、编辑和代理模式的统一人工智能界面

## 设置

Cursor 的设计灵活且可自定义。您可以通过两种方式对其进行配置：

### 光标设置

- 通过齿轮图标、`Cmd/Ctrl + Shift + J` 或命令调板 >`光标设置`进行访问
- 配置 AI 功能和光标特定首选项

### 编辑器设置

- 通过命令调板`（Cmd/Ctrl + Shift + P`）`>"首选项 "`访问`：打开设置（用户界面）`
- 调整编辑器行为和外观

让我们来详细了解每项功能：

### 制表符

Cursor 中的制表符补全由先进的人工智能模型提供支持，它能理解你的代码上下文。当你输入时，你将收到以下智能建议

- 完成当前代码行
- 建议整个函数的实现
- 帮助解决常见模式和模板问题
- 随着时间的推移适应你的编码风格

了解有关[Tab 功能的](https://docs.cursor.com/tab/overview)更多信息，或查看它[与 GitHub Copilot 的比较](https://docs.cursor.com/tab/from-gh-copilot)。

### 聊天

Cursor 提供统一的人工智能界面，三种模式可无缝协作：

**代理模式（默认）**

- 进行全代码范围的更改和重构
- 根据需求实现新功能
- 调试跨多个文件的复杂问题
- 生成测试和文档
- 保持整个项目的一致性

**提问模式**

- 询问有关特定代码部分的问题
- 获取复杂函数的解释
- 查找代码模式和示例
- 发现并了解您的代码库

**自定义模式**

- 创建适合自己工作流程的模式

在对话过程中切换模式，以最适合您当前的任务。了解有关[统一人工智能界面](https://docs.cursor.com/chat/overview)的更多信息，或探索[代理模式](https://docs.cursor.com/chat/overview)中的特定功能。

### 语境

上下文是 Cursor 所有人工智能功能的基础。其工作原理如下：

- 当您打开代码库时，我们会自动[索引您的代码](https://docs.cursor.com/context/codebase-indexing)，使其成为上下文。
- 使用[@ 符号](https://docs.cursor.com/context/@-symbols/overview)来精确控制所提供的上下文：
    - [@files](https://docs.cursor.com/context/@-symbols/@-files)和[@folders](https://docs.cursor.com/context/@-symbols/@-folders)表示特定路径
    - [@web](https://docs.cursor.com/context/@-symbols/@-web)用于外部文档
    - [@git](https://docs.cursor.com/context/@-symbols/@-git)用于版本控制上下文
- [为人工智能](https://docs.cursor.com/context/rules-for-ai)配置[规则](https://docs.cursor.com/context/rules-for-ai)以自定义行为
- 为外部上下文提供者设置[MCP](https://docs.cursor.com/context/model-context-protocol)
