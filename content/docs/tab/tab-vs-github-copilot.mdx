将 Cursor 的多行编辑和基于指令的补全与 GitHub Copilot 的单行插入进行比较

## Tab 的改进

Cursor 和 GitHub Copilot 最大的不同在于完成代码的方式。

GitHub Copilot 可以在Cursor位置插入文本。它不能编辑Cursor周围的代码或删除文本。

Cursor 可以在Cursor位置插入文本，而且功能更强大：

- 多字符编辑
- 基于指令的编辑

此外，Cursor 还能在上下文窗口中记录你最近的更改，因此它知道你下一步要做什么。

## 从 GitHub Copilot 移植

由于 Cursor 默认与 GitHub Copilot 一同安装，你可能同时安装了 GitHub Copilot 和 Cursor。建议在使用 Cursor 时关闭 GitHub Copilot。

默认情况下，Cursor 优先于 GitHub Copilot。如果你想使用 GitHub Copilot，可以在设置中[禁用 Cursor](https://docs.cursor.com/tab/overview#copilot-settings)。
